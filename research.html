Probabilistic Swarm Systems

As part of the preliminary paper “Verifying Emergence of Bounded Time Properties in Probabilistic Swarm Systems”, new semantics for probabilistic swarms has been explored and brought to light.

The parameterised semantics, in general, deals with the environment of the swarm as well as the individual agents. The environment’s state is changing depending on the actions performed by all the agents while each agent chooses their next actions based upon a probability distribution which is calculated based on the agent’s own current state.

A more restricted version, in terms of the bounding of the amount of time steps, of PCTL or Probabilistic real time Computation Tree Logic, is presented. This version, which is expressed in a fragment of the PCTL, aims to be a form of formal identification of emergent properties. The restricted PCTL is utilized to create an algorithm to solve the emergent property identification problem (which determines whether the specification is an emergent property of the system or not) and the emergence threshold identification problem (which computes an emergence threshold for a given property of the system).

Whether or not a global emergent behavior is exhibited in a swarm is the problem that is to be addressed. It is intended that a certain property will always be exhibited amongst the swarm for a number of agents above a threshold. This allows a swarm to be validated before deployment for an arbitrary number of agents – given that the global emergent behavior is to be checked amongst the agents. This will be achieved by the parameterised semantics described as above.

By the semantics introduced and the restricted algorith utilising PCTL, an unbounded swarm can be reasoned about which is a step towards the verification of unbounded drone swarms.

Synthesis of Fault Tolerance Ratio

The paper “Symbolic Synthesis of Fault-Tolerance Ratios in Parameterised Multi-Agent Systems” expands upon the verification of fault-tolerance properties of swarm systems.

To combat the uncertainty of possible impacts malfunctions may cause, the robustness of a system must be verified. A procedure is introduced which attempts to compute the maximal ratio of possible faulty agents for the specification, expressed in temporal logic, to always be upheld for an arbitrarily sized system. More than that, this allows for the establishment of a correct system according the fault tolerance of a given protocol.

Although the PIIS semantics and fault injection/tolerance concepts are similar, the new technique instead finds a maximal ratio of faulty agents, inside an error margin supplied, instead of a faulty to non-faulty ratio for a fixed population. As the number of agents are variable at any given point in time, a maximal ratio will be more relevant as it can be used to ensure the specification, in terms of the maximum number of faulty agents present, is always met no matter the size of the population.

The technique presented, adapted from CTL, symbolically derives the fault tolerance ratio (maximal ratio described above) and is incorporated with a general model checking algorithm to create a single procedure to determine the robustness of a multi-agent with varying population system altogether.
